# omniscope-tcr-r

> **Work in Progress** — This pipeline is unofficial, under active development, and not affiliated with Omniscope in any way. Use at your own risk.

R code for loading, filtering, and processing T-cell receptor (TCR) sequencing data generated by [Omniscope](https://www.omniscope.ai/). This repository shares the processing scripts as-is — it is **not** an R package.

The TCR sequencing technology and output format come from [Omniscope](https://www.omniscope.ai/). I have no affiliation with Omniscope and this code is an independent.

---

## Dependencies

Install the required R packages before use:

```r
install.packages("dplyr")
```

---

## How to Use

Source the script directly (no installation required):

```r
source("R/omniscope_tcr_r.R")
```

A full annotated example is available in [examples/run_example.R](examples/run_example.R).

---

## Input Files

The pipeline expects up to three CSV files from an Omniscope TCR run:

| File | Required columns |
|------|-----------------|
| **contigs** | `cell_id`, `contig_id`, `locus`, `v_call`, `j_call`, `cdr3`, `cdr3_aa`, `productive` |
| **counts** | `contig_id`, `counts` |
| **pairs** | `contig_id_A`, `cdr3_aa_A`, `contig_id_B`, `cdr3_aa_B` |

All three inputs accept a file path, a directory (CSV files are auto-discovered), or a pre-loaded `data.frame`.

## Example Workflow

```r
source("R/omniscope_tcr_r.R")

# --- Load and QC-filter TCR data ---
TCR.table.list <- load_omniscope_tcr(
  input.contigs = file.name.contigs,
  input.counts  = file.name.counts,
  input.pairs   = file.name.pairs,
  filter_nonproductive    = TRUE,
  filter_stop_codon       = TRUE,
  filter_out_of_frame     = TRUE,
  require_complete_vdj    = FALSE,
  remove_multi_chain      = FALSE,
  resolve_ambiguous_genes = "none",
  verbose                 = TRUE
)

# --- Retrieve outputs ---
# one row per cell × chain
df.cellwise   <- TCR.table.list$cellwise
# one row per unique contig
df.contigwise <- TCR.table.list$contigwise
# QC summary counts
df.qc         <- TCR.table.list$summary
```

---

## Outputs

`load_omniscope_tcr()` returns a named list with three elements:

| Element | Description |
|---------|-------------|
| `cellwise` | One row per cell × chain, after QC filtering |
| `contigwise` | One row per unique contig_id, with counts and pairing merged |
| `summary` | QC summary: raw/filtered contig counts, merge statistics |

---

## Function Reference

| Function | Description |
|----------|-------------|
| `load_omniscope_tcr()` | Main entry point — loads contigs, counts, and pairs; applies QC filters; returns both output tables |
| `load_omniscope_contigs_cellwise()` | Load and QC-filter contigs to a cell × chain table |
| `contigs_to_contigwise()` | Collapse cell-wise table to one row per unique contig |
